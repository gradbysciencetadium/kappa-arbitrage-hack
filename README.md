# Kappa Arbitrage — Sovereign, Auditable AI for UK Decisions

**FLock Sovereign AI Challenge — Track 3: AI Governance, Transparency & Trust.**

Kappa Arbitrage produces data-backed strategic analysis for high-stakes UK decisions
(beachhead: childcare site selection). What makes it a *governance* tool, not a chatbot:

- **The model writes, the code decides.** Every quantitative claim is computed
  deterministically in [src/data/derivations.js](src/data/derivations.js) from cited public
  data (Ofsted, ONS, IMD) — never generated by the LLM. No fabricated figures.
- **Sovereign inference.** Analysis is routed through **FLock's** OpenAI-compatible API
  (`qwen3-30b`) — open, locally-governable inference, not a foreign proprietary black box.
- **Tamper-evident accountability ledger.** Every analysis is recorded as a SHA-256
  **hash-chained** entry (inputs hash → public sources → deterministic outputs → FLock model id →
  prediction → predicted-vs-actual). Any after-the-fact edit is detectable and pinpointed live.
- **Disclosed limitations.** Sample data is labelled `SAMPLE` on screen; the backtest accuracy
  record is shown honestly (including a logged miss).

```
Browser ─► /api/chat   ─► Kappy (intake)            ── FLock sovereign inference (qwen3-30b)
Browser ─► /api/analyze ─► Bara: plan → data-blind critic → DETERMINISTIC metrics
                              → grounded workers → validation → synthesis → judge → report
                              └─► AI Accountability Ledger (hash-chained, /governance.html)
Browser ─► /api/ledger, /api/ledger/verify[?tamper=1]   ── live verify + tamper demo
```

## Setup
1. `npm install`
2. Copy `.env.example` to `.env` and set keys (see below).
3. `npm start` → open http://localhost:3000 (consultation) and `/governance.html` (audit ledger).

## Keys (.env)
- `GEMINI_API_KEY` — default inference (https://aistudio.google.com/apikey).
- **Sovereign mode:** `SOVEREIGN_AI=1`, `FLOCK_API_KEY`, `FLOCK_MODEL=qwen3-30b-a3b-instruct-2507`
  — routes **every** pipeline stage through FLock. Verified end-to-end (~35s/report).
- **Persistence (optional):** `SUPABASE_URL` + `SUPABASE_KEY`; run [supabase/schema.sql](supabase/schema.sql).
  Without these the app uses an in-memory fallback (state resets on restart).
- The LLM layer is model-agnostic ([src/llm/models.config.js](src/llm/models.config.js)) — any
  stage can be routed to FLock, Gemini, Groq, or OpenRouter by config.

## The governance artifact
`/governance.html` shows the **AI Accountability Ledger**: live hash-chain integrity, the
backtest accuracy record (predicted vs. actual for known openings), the sovereign model used,
and a **"Verify chain" / "Simulate tamper"** demo — editing one entry turns its row red and the
chain reports the break. Maps to EU AI Act Art. 12/19 automatic-logging obligations.

## Honest scope
Tamper-*evident*, not tamper-*proof* (roadmap: anchor the head hash to an external notary /
on-chain). Backtest is a directional-agreement check on labelled comparables, growing with
engagements. Childcare demo data is a disclosed sample.
